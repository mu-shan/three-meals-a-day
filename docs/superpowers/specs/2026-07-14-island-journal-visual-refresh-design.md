# 岛屿手账视觉改版设计

## 背景

当前版本已经具备三餐随机、单餐换菜、单菜换菜、喜欢与不喜欢、采购清单、偏好管理和本地备份等完整功能，但视觉上存在以下问题：

- 首屏大型家庭表情插画占据过多空间，用户进入页面后无法立即看到菜单。
- 大面积亮黄色、粗描边、重阴影和不规则边框同时出现，页面显得拥挤且偏幼稚。
- 自定义装饰样式过多，虽然使用了 `animal-island-vue`，整体组件语言仍不统一。
- 首页、偏好页和底部导航的视觉层级不够一致。

本次改版不调整随机菜单、偏好权重、本地存储和备份恢复的业务规则，只重构页面布局、视觉系统和交互层级。

## 设计目标

1. 采用“岛屿手账”方向，保留《动物森友会》风格的自然、松弛和手作感。
2. 整体气质从儿童插画转为温暖、成熟、适合长期使用的家庭餐桌工具。
3. 手机第一屏同时呈现日期、问候、核心操作和第一顿饭的部分内容。
4. 优先使用 `animal-island-vue` 的现成组件，让页面风格由统一的组件体系支撑。
5. 所有现有功能保留，但允许重新安排按钮和信息位置。
6. 桌面端继续展示居中的手机宽度画布，不新增独立桌面布局。

## 视觉方向

### 色彩

- 主背景：沙滩米黄和浅纸张色，降低当前高饱和亮黄色的面积。
- 主操作色：森林绿，用于主要按钮、选中状态和当前进度。
- 强调色：陶土橙，用于喜欢状态、提醒和小面积重点信息。
- 文字色：深棕，不使用纯黑。
- 辅助色：鼠尾草绿、浅木色和奶油白。

色彩应以低饱和、大面积浅色为主。森林绿和陶土橙只用于建立层级，不平均铺满页面。

### 字体

- 页面主标题和章节标题使用项目环境可用的中文宋体或衬线字体，形成家庭餐桌杂志感。
- 正文、按钮和数据继续使用清晰的中文无衬线字体。
- 不再使用超大、过度活泼的标题字体。

### 材质与装饰

- 背景保留非常轻的纸张纹理和手账横线。
- 每个主要区块最多使用一种手作装饰，例如胶带、虚线分隔或不规则圆角，不能同时叠加。
- 移除大型人物、动物和 Twemoji 家庭插画。
- 森友会氛围通过纸张、木色、叶片、圆润轮廓和组件库动效表达，不直接使用游戏角色或官方素材。

## 首页结构

首页从上到下分为以下区域：

1. 日期与时间
2. 每日问候和页面标题
3. 今日用餐进度
4. 重新安排今日菜单
5. 早餐、午餐、晚餐卡片
6. 今日采购清单
7. 森林风格页脚
8. 固定底部导航

### 日期与时间

- 左侧显示页面自行格式化的中文日期和星期。
- 右侧使用 `Time` 组件显示实时钟表。
- `Time` 当前没有配置项且自带英文日期，因此通过局部样式只保留钟表区域，不让英文日期与中文界面冲突。
- 对组件内部类名的样式覆盖集中在一个明确位置，并由组件测试保护，避免升级依赖时静默失效。

### 问候与标题

- 使用 `Typewriter` 显示“早上好”“中午好”“傍晚好”等根据当前时间变化的每日问候。
- 打字效果在每次进入首页时播放一次，不随菜单重新随机而重复。
- 用户开启“减少动态效果”时直接显示完整文本。
- 主标题为“一家人的三顿饭”，字号明显小于当前首屏大标题。

### 今日用餐进度

组件库当前最新版本 `animal-island-vue@0.2.1` 没有导出 `Progress`，因此新增项目内的 `MealProgress` 组件。

进度条包含早餐、午餐、晚餐三个节点，并沿用现有餐次时间 `07:30`、`12:00`、`18:30` 作为阶段边界：

- `00:00-07:29`：三个节点均未完成，早餐为当前节点。
- `07:30-11:59`：早餐完成，午餐为当前节点。
- `12:00-18:29`：早餐和午餐完成，晚餐为当前节点。
- `18:30-23:59`：三个节点均完成，晚餐保留为最近节点但不再播放呼吸动效。

进度只表达一天中的时间阶段，不表示用户真的已经吃完某顿饭，也不伪造菜单生成百分比。

### 核心操作

- “重新安排今日菜单”是首页唯一的大型主按钮，继续触发整日菜单随机。
- “口味偏好”不再出现在首屏按钮区，只保留固定底部导航入口。
- 每顿饭的“换一餐”和每道菜的替换、喜欢、不喜欢继续放在对应卡片内部。

### 三餐卡片

- 使用 `Card` 承载早餐、午餐和晚餐。
- 使用 `Title` 显示餐次标题，使用 `Divider` 分隔卡片标题和菜品。
- 使用 `Button` 统一整日、单餐和单菜替换操作。
- 菜品继续显示名称、主要食材、角色标签和喜欢状态。
- 保留至少 44px 的触控区域。
- 午餐不再通过明显上移和重阴影强调，三餐使用一致结构，仅通过轻量色彩区分。

### 采购清单

- 使用 `Collapse` 按蔬菜、肉蛋奶、主食和水果折叠分类，减少手机页面长度。
- 默认展开第一项有内容的分类，其余分类折叠。
- 分类标题显示食材数量，展开后显示完整清单。
- 使用 `Icon` 中适合的购物图标作为区块标识；不为了使用组件而勉强映射无关图标。

### 页脚

- 使用 `Footer` 的 `tree` 类型替换当前文字页脚。
- 页脚之前保留一句简短文案：“今天也要和喜欢的人，好好吃饭。”

## 口味偏好页

偏好页与首页共用同一色板、字体、纸张纹理、边框和底部导航。

### 页面头部

- 使用较紧凑的标题区，不再使用大型独立宣传卡片。
- 展示“喜欢”“不喜欢”和总记录数三个统计数据。
- 使用 `Title` 和 `Divider` 建立层级。

### 偏好列表

- 使用 `Tabs` 切换“喜欢的菜”和“不喜欢的菜”。
- 喜欢列表保留取消喜欢操作。
- 不喜欢列表保留恢复到随机菜单操作。
- 列表项使用轻量 `Card`，菜品图片、名称、食材和操作保持清晰。
- 空状态使用组件库的视觉语言，但不新增大型角色插画。

### 备份与恢复

- 使用 `Collapse` 收起“数据备份与恢复”，默认关闭，降低页面首屏负担。
- 导出、选择文件、预览和确认恢复的业务流程不变。
- 导入校验错误继续在面板内部显示，不使用静默失败。

## 组件映射

| 组件 | 使用位置 | 说明 |
| --- | --- | --- |
| `Time` | 首页顶部 | 只展示实时钟表，中文日期由页面控制 |
| `Typewriter` | 首页问候 | 每次进入首页播放一次，支持减少动态效果 |
| `Title` | 三餐、采购清单、偏好页 | 统一章节标题 |
| `Divider` | 主要章节和餐次卡片 | 统一分隔节奏 |
| `Card` | 三餐、菜品、偏好列表 | 减少自定义厚重容器样式 |
| `Button` | 随机、换菜、恢复等操作 | 统一操作状态和尺寸 |
| `Tabs` | 偏好分类 | 替换自定义双标签按钮 |
| `Collapse` | 采购清单、备份恢复 | 缩短移动端长页面 |
| `Icon` | 导航和采购区块 | 只使用语义匹配的图标 |
| `Loading` | 菜单随机中 | 保留现有加载反馈 |
| `Footer` | 页面底部 | 使用 `tree` 类型强化岛屿氛围 |
| `MealProgress` | 首页顶部 | 项目自定义三餐时间进度 |

以下组件不纳入本次改版：

- `Tooltip`：移动端需要额外点击，当前操作已有明确文案。
- `Phone`：只适合组件演示，不应嵌套在真实手机页面中。
- `CodeBlock`、`Table`：与当前业务无关。
- `Cursor`：移动端没有价值。

## 状态与数据流

本次改版不改变现有 Pinia store 和本地存储结构。

- 首页继续从菜单 store 读取今日菜单、随机状态、采购清单和反馈文案。
- 偏好页继续从偏好 store 读取喜欢和不喜欢列表。
- `MealProgress` 只接收当前 `Date` 或当前时间阶段，不写入 store 和本地存储。
- `Typewriter` 的播放状态属于页面展示状态，不持久化。
- 跨日刷新逻辑继续由现有菜单 store 处理。

## 动效

- 每次进入页面时使用一次轻量的分段出现动画。
- `Typewriter` 只播放每日问候。
- 当前用餐节点使用缓慢呼吸动效。
- 菜单随机时继续使用 `Loading`，卡片只做淡出和恢复，不使用大幅旋转。
- 所有动画在 `prefers-reduced-motion: reduce` 下停用或缩短到近乎即时。

## 响应式与可访问性

- 页面最大宽度维持 560px，桌面端居中展示。
- 支持 320px 及以上手机宽度。
- 固定底部导航继续适配 `safe-area-inset-bottom`。
- 交互元素触控区域不小于 44px。
- 颜色对比不能只依赖浅色背景，当前状态同时使用文字或形状表达。
- 图标按钮必须提供可读的 `aria-label`。
- 随机菜单和偏好反馈继续使用状态区域向辅助技术播报。

## 错误和边界处理

- 菜品图片加载失败继续使用本地占位图。
- 备份导入继续执行严格校验和原子恢复。
- 当系统时间不可用时使用浏览器提供的当前时间，不新增远程时间服务。
- `MealProgress` 只基于本地小时计算阶段，不处理用户是否实际用餐。
- 如果组件库内部结构在未来升级后变化，Time 组件的样式测试应及时暴露问题。

## 测试策略

- 保留现有 store、备份、路由和菜单生成测试。
- 更新组件测试以适配重排后的按钮和标题，但继续使用稳定的 `data-testid` 查询关键交互。
- 为 `MealProgress` 增加 `07:29`、`07:30`、`12:00`、`18:30` 等边界测试。
- 为首页增加 Time、Typewriter、三餐进度和主操作存在性测试。
- 为偏好页增加 Tabs 切换和备份 Collapse 展开测试。
- 验证减少动态效果下问候文本可立即读取。
- 执行单元测试与类型检查；只有用户明确要求时才执行生产构建和浏览器页面验证。

## 不在本次范围

- 不修改菜品库、随机算法和喜欢权重。
- 不新增登录、云同步或跨设备同步。
- 不新增独立桌面端布局。
- 不直接复制任天堂角色、图像或官方素材。
- 不升级 `animal-island-vue`，当前 npm 最新版仍为 `0.2.1`。

## 验收标准

- 手机第一屏能看到中文日期、实时钟表、每日问候、三餐进度、整日随机按钮和早餐内容。
- 页面不再出现大型家庭表情插画。
- 首页和偏好页使用一致的岛屿手账视觉系统。
- 现有随机、换菜、喜欢、不喜欢、采购清单和备份恢复功能全部可用。
- `Time`、`Typewriter`、`Title`、`Divider`、`Card`、`Button`、`Tabs`、`Collapse`、`Loading` 和 `Footer` 在有明确业务用途的位置使用。
- 自定义 `MealProgress` 能根据当前本地时间正确显示三餐阶段。
- 320px 宽度下无横向滚动，主要操作满足 44px 触控尺寸。
- 现有测试和新增测试通过，类型检查通过。
