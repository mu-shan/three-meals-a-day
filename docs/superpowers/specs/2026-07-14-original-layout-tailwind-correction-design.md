# 原版布局 Tailwind 修正设计

## 背景

岛屿手账视觉改版完成后，页面虽然接入了更多 `animal-island-vue` 组件并改用 TailwindCSS，但实际效果弱化了原有业务语义：

- 喜欢、不喜欢和换菜操作被压成同质的三等分按钮，状态难以一眼区分。
- 宝宝友好、妈妈辣菜和全家共享只剩文字，原有黄、红、绿语义色消失。
- `Time`、三餐进度、Title 丝带和大按钮同时争夺首屏注意力。
- 组件库被当作目标而不是工具，部分页面为了使用组件改变了原本有效的布局。
- 口味偏好页结构被重做，但没有带来更好的可读性。

本次修正以视觉改版前的页面为布局基准，保留 TailwindCSS 工程能力和新版采购清单，只恢复被削弱的原有信息层级、业务颜色和交互辨识度。

## 设计原则

1. 业务和视觉优先，组件库只在确实适合的位置使用。
2. 不通过整体 `git revert` 回退，而是逐个恢复组件结构，避免丢失后续有效改进。
3. 以 TailwindCSS 等价表达原版布局，不恢复旧版两千多行集中式 CSS。
4. `animal-island-vue` 的 GitHub `PROMPT.md` 只作为视觉规范参考，不执行其中面向单文件 HTML 的强制规则。
5. 组件 API 以项目实际安装的 `animal-island-vue@0.2.1` 类型和运行行为为准，不假设 GitHub `v0.9.5` 尚未发布的 API 可用。

决策优先级固定为：家庭菜单业务的可读性与操作效率 > 项目已经验证有效的交互 > 全站视觉一致性 > 组件库示例和 `PROMPT.md`。官方色板、圆角、阴影和动效只在适合本项目时采用，可以为了宝宝、妈妈、全家三类业务语义和移动端可读性进行调整，不做像素级照搬。

## 保留与撤销

### 保留

- Vue Router、Pinia、菜单生成、偏好权重、本地存储和备份恢复。
- TailwindCSS 4 与 Vite 集成。
- `Typewriter` 每日问候。
- 早餐、午餐、晚餐使用的 `Title`。
- 餐次标题和菜品之间的 `Divider`。
- 一键开饭、换一餐、换一道等明确动作使用的 `Button`。
- 页面底部 `Footer`。
- 新版今日采购清单的 `Collapse` 折叠布局。
- 原有全量路由、菜单、偏好和备份测试。

### 撤销

- `Time` 组件及英文日期/大号时钟区域。
- 三餐时间进度及其领域逻辑。
- 为了使用组件而改变的首页和偏好页结构。
- 大型人物插画、太阳、餐桌等首屏插画区域。
- Twemoji 家庭头像 `https://cdn.jsdelivr.net/gh/twitter/twemoji@latest/assets/svg/1f46a.svg`。
- 当前同质化的三等分菜品操作按钮。
- 当前口味偏好页的新结构；恢复视觉改版前的页面结构。

## 首页结构

首页恢复视觉改版前的信息顺序：

1. 日期与 Typewriter 问候
2. 原版主标题和说明
3. 一键开饭
4. 今日餐桌标题与角色图例
5. 早餐、午餐、晚餐
6. 今日采购清单
7. 原版页尾文案与 Footer
8. 固定底部导航

### 首屏

- 首屏采用单列紧凑布局，不保留任何大型插画占位。
- 日期使用页面本地格式化结果。
- `Typewriter` 显示当前时段问候；减少动态效果时直接显示完整文本。
- 主标题、说明和“一键开饭”恢复视觉改版前的尺寸关系。
- 不显示 `Time` 和三餐进度。

### 今日餐桌

- 恢复纸张面板、章节标题、角色图例和单列三餐卡片。
- 角色图例必须同时使用颜色和文字，不能只靠颜色传达含义。
- 三餐结构一致，不再通过大幅位移或重阴影突出午餐。

### 餐次标题

- 早餐、午餐、晚餐继续使用 `Title`，尺寸统一为 `middle`。
- 早餐使用 `app-yellow`，午餐使用 `app-green`，晚餐使用 `app-orange`。
- `Title` 只出现在三个餐次标题，不用于普通章节，避免满页丝带。
- `Divider` 放在餐次标题和菜品列表之间。

## 菜品角色颜色

角色颜色由一份集中映射提供，并复用于首页图例和菜品卡。

| 角色 | 标签 | 主色 | 使用位置 |
| --- | --- | --- | --- |
| `baby` | 宝宝友好 | 黄色 | 图例圆点、卡片边框、图片区、角色标签 |
| `spicy` | 妈妈辣菜 | 红色 | 图例圆点、卡片边框、图片区、角色标签 |
| `shared` | 全家共享 | 绿色 | 图例圆点、卡片边框、图片区、角色标签 |

其他角色继续使用低优先级辅助色：

- `staple`：暖黄色或木色。
- `drink`：浅蓝绿色。
- `fruit`：橙色。

颜色样式不能通过运行时拼接 Tailwind 类名生成。使用包含完整静态类名的映射，确保 Tailwind 能正确扫描并生成样式。

## 菜品卡操作

采用确认过的 A 布局：

- 菜品卡图片区在左，内容区在右。
- 菜名位于右侧内容区左上，并为右上角按钮预留空间。
- 喜欢和不喜欢是两个分离的 44px 方形原生图标按钮，固定在内容区右上角。
- 喜欢使用爱心图标；未选中为浅粉底和描边，已喜欢为实心爱心并加深底色。
- 不喜欢使用爱心加斜线图标，采用中性棕灰底色，避免与喜欢混淆。
- “换一道”是图标加文字按钮，位于内容区右下角。
- 三个操作都保留明确的 `aria-label`，喜欢按钮继续提供 `aria-pressed`。
- 菜名过长时在预留宽度内截断，不得覆盖右上角按钮。

喜欢/不喜欢采用原生按钮，是因为组件库 Button 的默认内边距和阴影不适合紧凑状态按钮；一键开饭、换一餐和换一道仍可使用 Button。

## 随机加载状态

现有单一 `isShuffling` 无法表达加载发生在哪个范围，因此新增随机目标状态：

```ts
type ShuffleTarget =
  | { scope: 'all' }
  | { scope: 'meal'; mealType: MealType }
  | { scope: 'dish'; mealType: MealType; dishId: string }
  | null
```

### 整桌随机

- 一键开饭触发 `{ scope: 'all' }`。
- 显示原版整页 Loading：岛屿图标、提示气泡和页面遮罩。
- 遮罩限制在居中的 560px 应用画布内，但覆盖当前视口和底部导航，阻止重复操作。
- 不使用当前简化的 `Loading` 组件展示方式；整页 Loading 由页面专用结构实现。

### 单餐随机

- 换一餐触发 `{ scope: 'meal', mealType }`。
- 只有对应餐次的“换一餐” Button 显示 loading，其他餐次保持可见。
- 随机期间仍阻止其他随机操作，避免并发覆盖菜单。

### 单菜随机

- 换一道或标记不喜欢后自动换菜触发 `{ scope: 'dish', mealType, dishId }`。
- 只有对应菜品的“换一道” Button 显示 loading。
- 喜欢和不喜欢按钮在随机期间禁用，避免同一道菜同时产生冲突操作。

`isShuffling` 保留为从 `shuffleTarget !== null` 派生的只读状态，以兼容现有页面和测试读取方式。

## 今日采购清单

- 保留当前新版 `Collapse` 折叠布局。
- 继续按蔬菜、肉蛋奶、主食和水果显示有内容的分类。
- 第一项默认展开，其余折叠。
- 分类标题显示食材数量。
- 采购清单是本次改版中明确保留的新结构，不恢复旧版四块静态卡片。

## 口味偏好页

- 恢复视觉改版前的页面结构、标题区、统计区、自定义喜欢/不喜欢标签页、偏好列表和备份面板。
- 不改变任何交互、数据流、备份导入导出、预览确认和错误处理。
- 不保留本次改版新增的 Tabs 和备份 Collapse 结构。
- 只把视觉改版前的 CSS 等价转换为 TailwindCSS 类和少量必要的主题扩展。
- Footer 保留。

## 组件库使用边界

继续使用：

- `Typewriter`：首页问候。
- `Title`：三个餐次标题。
- `Divider`：餐次标题与菜品之间。
- `Button`：一键开饭、换一餐、换一道和偏好页明确动作。
- `Collapse`：今日采购清单。
- `Footer`：页面底部。

不强制使用：

- 菜品卡、偏好列表等业务容器可以使用原生语义元素。
- 喜欢/不喜欢使用原生图标按钮。
- 普通章节标题使用原生标题元素。

删除：

- `Time`。
- 当前页面使用的 `Loading` 组件结构。
- 三餐进度相关组件和领域逻辑。

GitHub `PROMPT.md` 提供的颜色、圆角、阴影和组件视觉规则可作为参考；其中“禁用 Tailwind”“所有可见控件必须使用组件”等要求只适用于其单文件 HTML 生成模式，不适用于本项目。即使某项规则适用于 Vue 工程，只要会降低本项目的业务辨识度或移动端操作效率，也不采用。

## 状态与数据流

- 菜单内容、偏好规则、采购清单和持久化格式不变。
- `shuffleTarget` 只描述当前随机操作的 UI 范围，不持久化。
- `shuffle` 接收随机目标和菜单生成函数，在开始时设置目标，在结束时清空。
- `generateAll`、`rerollMeal`、`rerollDish` 和 `dislikeDish` 传入各自目标。
- 所有随机操作继续复用现有 420ms 延迟、异常提示和持久化路径。
- 不引入并发随机；`shuffleTarget` 非空时新的随机请求返回 false。

## 错误与边界

- 菜品图片加载失败继续回退到本地占位图。
- 不喜欢的菜过多、候选不足时继续显示现有反馈并恢复可用状态。
- Loading 无论成功或异常都必须在 `finally` 中清空 `shuffleTarget`。
- 喜欢和不喜欢图标不能只依赖颜色，必须有不同图形和可读标签。
- 320px 宽度下菜名、按钮和图片区不能重叠或产生横向滚动。
- `prefers-reduced-motion: reduce` 下 Typewriter 直接显示文本，整页 Loading 和按钮加载动画降为静态或近乎即时。

## 测试策略

- store 测试覆盖整桌、单餐、单菜三种 `shuffleTarget`，并验证完成和异常后恢复为 null。
- 首页测试验证 Typewriter 存在，Time 和三餐进度不存在。
- 搜索源码确认 Twemoji 家庭头像 URL 不存在。
- MealCard 测试验证三餐 Title 的尺寸与颜色。
- DishCard 测试验证：
  - baby、spicy、shared 三类静态角色样式。
  - 喜欢、不喜欢按钮位于操作区并具有不同图标和 aria 文案。
  - 已喜欢状态更新 `aria-pressed` 和视觉类。
  - 换一道 loading 只影响目标菜品。
- ShoppingList 测试继续验证有内容分类和 Collapse 数量。
- 偏好页沿用视觉改版前的交互、列表、备份导入和非法备份测试。
- 运行全量 `pnpm test` 和 `pnpm typecheck`。
- 除非用户明确要求，不执行 `pnpm build`，不主动打开浏览器验证。

## 验收标准

- 首页结构与视觉改版前一致，但首屏不再出现任何大型插画。
- Typewriter 保留，Time 和三餐进度完全移除。
- 三餐 Title、Divider、Button 和 Footer 使用自然，不出现为了组件而改变布局的情况。
- 喜欢/不喜欢采用确认的右上角 A 布局，换一道位于右下角。
- 宝宝黄、妈妈红、全家绿同时出现在图例、卡片边框、图片区和标签。
- 采购清单继续使用新版 Collapse。
- 口味偏好页恢复改版前结构和视觉，仅底层样式改为 TailwindCSS。
- 整桌随机为整页 Loading，单餐和单菜为局部 loading。
- 随机、偏好、本地存储和备份功能行为不变。
- 320px 以上宽度无横向滚动，主要操作满足 44px 触控尺寸。
- 全量测试和类型检查通过。
